# Zero-Employee Orchestrator — 为什么需要这个系统

> 从"使用"AI 的时代，迈向让 AI"为你工作"的时代。

---

## 你的 AI 使用方式是否遇到了这些问题？

"每次向 ChatGPT 提问，都要重新输入相同的背景信息。"

"手动复制 AI 的回答，然后粘贴为下一步的输入。"

"AI 给出的答案是否正确，每次都要自己逐一确认。"

"昨天让 AI 做的任务进展到哪里了，完全无法掌握。"

"让 AI 做某件事时，担心它会误删重要数据或发送错误的邮件。"

这些都是 AI 聊天工具的**结构性局限**。Zero-Employee Orchestrator 就是为了从根本上解决这些问题而诞生的。

---

## 什么是 Zero-Employee Orchestrator？

**从"一个人使用的工具"到"让 AI 作为组织运转的基础设施"**

Zero-Employee Orchestrator 是一个 **AI 业务编排平台**，只需用自然语言下达指令，多个 AI 代理即可组队，自主完成计划、执行、验证和改进。

它不仅仅是一个自动化工具，而是一个全新的平台，**将整个 AI 组织引入你的业务**。

---

## 与现有系统的对比

### 与其他 AI 代理（AutoGPT、CrewAI 等）的区别

| 方面 | 其他 AI 代理 | Zero-Employee Orchestrator |
|------|-------------|---------------------------|
| **多 AI 协作** | 有限（以单一代理为中心） | 分工明确的 AI 团队并行执行 |
| **记忆** | 仅在运行时（结束即丢失） | 持久化记忆经验、模式和失败 |
| **任务管理** | 仅在执行期间跟踪 | 通过 Spec / Plan / Tasks 结构化存储 |
| **质量保证** | 无或单模型 | Judge Layer 两阶段 + Cross-Model Verification |
| **危险操作防护** | 无（全自动执行） | 通过审批流程强制阻断 |
| **故障恢复** | 停止或简单重试 | Self-Healing DAG 自动重新规划 |
| **审计日志** | 无或有限 | 全操作追踪记录 |
| **成本管理** | 无 | 实时追踪 Token 消耗和预算 |
| **可扩展性** | 需要修改代码 | 通过 Skill / Plugin / Extension 灵活扩展 |

### 与 RPA / 传统业务自动化工具的区别

| 方面 | 传统 RPA | Zero-Employee Orchestrator |
|------|---------|---------------------------|
| **配置难度** | 需要手动设计流程/脚本 | 只需用自然语言下达指令 |
| **应对变化** | 需要手动修改流程 | AI 自主重新规划 |
| **非常规任务** | 难以应对 | 使用自然语言即可处理 |
| **判断与推理** | 基于规则 | 通过 LLM 进行高级推理 |
| **审批流程** | 需要手动设置 | 自动检测危险操作并请求审批 |

### 与 n8n / Make（低代码自动化）的区别

| 方面 | n8n / Make | Zero-Employee Orchestrator |
|------|-----------|---------------------------|
| **任务定义** | 在 GUI 中拖放节点 | 一句自然语言 |
| **动态重新规划** | 静态流程 | 故障时自动重建 DAG |
| **与 AI 的深度集成** | 作为 API 模块添加 | AI 本身就是编排器 |
| **质量验证** | 无 | Cross-Model Verification |

## 本系统的优势

### 1. "一声令下"，AI 组织即刻运转

```
"制作本月的社交媒体发布日历，并汇总每篇帖子的文案。"
```

只需输入这句话：
1. AI 深入挖掘需求（Design Interview）
2. 自动分解任务并安排并行调度（DAG）
3. 各 AI 代理执行分配的任务
4. Judge Layer 验证质量和安全性
5. 发布前必定请求你的批准

人工操作仅需"点击批准按钮"。

---

### 2. 安全性从架构层面内置

将工作委托给 AI 时，最大的顾虑是"它可能会失控"。

Zero-Employee Orchestrator 在**架构层面解决了这个问题**：

```
外部发送 → 必须经过审批对话框
文件删除 → 必须经过审批对话框
计费操作 → 必须经过审批对话框
权限变更 → 必须经过审批对话框
```

危险操作被物理性地阻断，AI 无法直接执行。

此外，**Judge Layer** 对 AI 输出进行两阶段验证：
- **Stage 1**：基于规则的快速检查（检测禁止操作和凭证泄露）
- **Stage 2**：由不同模型进行 Cross-Model Verification（相互确认回答质量）

---

### 3. AI"从经验中学习"

传统 AI 聊天在对话结束后就会忘记一切。

Zero-Employee Orchestrator 通过 **Experience Memory** 实现：
- 记忆和复用过去的成功模式
- 分类失败原因（Failure Taxonomy）并防止复发
- 每次执行任务都变得更智能

重复执行同类业务的次数越多，AI 团队的准确性和速度就越高。

---

### 4. Self-Healing — 失败了也不会停下

复杂任务可能在执行过程中失败。

传统自动化工具的结果是"发生错误，已停止"。

Zero-Employee Orchestrator 的 **Self-Healing DAG**：
1. 检测故障
2. 分析原因
3. 自动生成替代方案（Re-Propose Layer）
4. 重建 DAG 并继续执行

无需人工干预，即可自主规避大量障碍。

---

### 5. 完全的透明性与可审计性

"不知道 AI 做了什么"是企业采用 AI 的一大障碍。

Zero-Employee Orchestrator 将所有重要操作记录在审计日志中：
- **谁**发出了什么请求
- **哪个 AI 模型**执行了操作
- **花费了多少**成本
- **输出了什么**内容
- **谁**批准了操作

完全满足合规、内部审计和成本管理的需求。

---

### 6. 现在就可以免费开始

AI 工具的导入障碍之一是"获取 API 密钥和设置计费"。

Zero-Employee Orchestrator 提供 **3 种免费使用方式**：

1. **Google Gemini 免费 API（推荐）**：无需信用卡即可获取 API 密钥，运行稳定
2. **Ollama（本地 LLM）**：将模型下载到本地电脑，完全免费、离线、无限制使用
3. **订阅模式（Subscription Mode）**：无需 API 密钥即可立即使用（适合试用，稳定性较低）

最平衡的入门方式：
```env
GEMINI_API_KEY=<从 Google AI Studio 获取的密钥>
DEFAULT_EXECUTION_MODE=free
```

正式运营时，通过设置 OpenRouter / OpenAI / Anthropic 的 API 密钥，即可使用 Claude Opus 4.6、GPT-5.4、Gemini 2.5 Pro 等高质量模型。支持的模型在 `model_catalog.json` 中动态管理，无需修改代码即可添加或切换模型。

---

### 7. AI 分身与 AI 秘书 — 你的"替身"在工作

**AI 分身（AI Avatar Plugin）** 学习你的决策标准和写作风格，以"你的替身"身份运作：
- 在 Judge Layer 的质量评估中反映你的价值观
- 以你的语气创建草稿
- 根据你的标准判断任务优先级

**AI 秘书（AI Secretary Plugin）** 是连接 AI 组织与你的"枢纽"：
- 每日早间简报（待批准事项、进行中任务、日程安排）
- 带优先级的下一步行动建议
- 通过 Discord / Slack / LINE 发送通知和操作

两者都是以 Plugin 形式安装的，如果不需要可以不安装。

---

### 8. 从聊天工具指挥 AI 组织

安装 Discord / Slack / LINE Bot Plugin 后，就可以从日常使用的聊天应用直接操作。

```
/zeo ticket 制作竞品分析报告
/zeo briefing
/zeo approve 12345
```

无需打开 Web 仪表板，在聊天的延伸中即可驱动 AI 组织。

---

### 9. 三层扩展体系定制你的业务

与其从零开始构建能处理所有业务的通用 AI，不如**组建专业化的 AI 团队**，效率要高得多。

```
Skill     — 单一用途的处理（例：网页抓取、翻译、分析）
Plugin    — 业务功能包（例：AI 分身、AI 秘书、YouTube 运营、Discord Bot）
Extension — 连接和 UI 扩展（例：Obsidian 集成、MCP、通知）
```

只需用自然语言描述技能，AI 就会自动生成技能代码。无需专业编程知识。

## 设计理念

Zero-Employee Orchestrator 基于以下信念设计：

> **"AI 不是工具，而是组织的一员。"**

不是一次性的问答，而是理解业务背景、制定计划、团队协作、从失败中学习、与人类建立恰当的关系——这才是真正的 AI 活用。

为此：
- **绝不取消人类的最终决定权** — 危险操作必须经过审批
- **绝不制造黑箱** — 所有操作可视化与记录
- **持续改进** — 经验记忆、失败分类、质量提升
- **保持可扩展** — Skill / Plugin / Extension 的明确边界
