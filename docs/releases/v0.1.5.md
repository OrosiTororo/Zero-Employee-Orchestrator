### New â€” Copilot Cowork-Inspired Task Management

Inspired by Microsoft Copilot Cowork's plan-approve-execute-steer model:

- **Plan Preview**: Dispatch tasks with `preview_only=true` to see the execution plan before committing. Review steps, then approve via "Approve & Execute" button
- **Needs Input Checkpoint**: Tasks auto-detect when LLM requests human input and pause with a reason message. Users provide input through the DispatchPage or CLI to resume
- **Mid-Execution Steering**: Redirect running tasks with additional instructions without cancelling them
- **DispatchPage Redesign**: Active/Done/All tabs, expandable task cards with plan steps, steering/resume inputs, color-coded status badges (Queued, Running, Needs Input, Done, Failed, Preview)
- **Rich Approval Preview**: Approval cards now show what the action will do, risk-level color badges (green/yellow/orange/red), and task context
- **CLI Task Management**: 6 new slash commands: `/dispatch`, `/tasks`, `/status`, `/approve`, `/reject`, `/cancel`

### Critical Fixes

**1. Desktop Auto-Update (CRITICAL)**

Auto-updates were completely broken for all v0.1.2â€“v0.1.4 users. Three independent root causes fixed:

- **Release workflow jq bug** â€” `latest.json` platforms object was always empty
- **CSP blocking updater** â€” `connect-src` only allowed `api.github.com`, not `github.com`
- **Manual-only installation** â€” Now auto-downloads and installs by default (configurable in Settings)

**2. Judge Severity Bug (CRITICAL)**

LLM gateway errors returned `CompletionResponse(content="Error: ...")` which passed the Judge quality check because the `no_error_response` rule had `severity="warning"` (score 0.95 > threshold 0.6). Changed to `severity="error"` and added `JudgeVerdict.FAIL` check to executor. Tasks now correctly fail when LLM calls fail.

**3. Frontend Path Mismatch**

DashboardPage and AgentMonitorPage called non-existent `/approvals?status=requested` instead of the correct company-scoped `/companies/{cid}/approvals`.
