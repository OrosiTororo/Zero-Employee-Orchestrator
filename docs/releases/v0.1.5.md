# v0.1.5 Release Notes

> Release date: 2026-04-07

## Desktop Auto-Update Fix & Release Pipeline Hardening

This release fixes a critical bug that prevented desktop auto-updates from working for all users who installed v0.1.2–v0.1.4. Three independent root causes were identified and fixed.

### Critical Fixes

**1. Release Workflow — Empty `latest.json` Platforms (CRITICAL)**

The `merge-updater-json` job in the release workflow had a jq operator precedence bug that caused the merged `latest.json` to always have an empty `platforms` object. This meant that even though the updater endpoint existed, clients would find no download URLs for any platform — causing a silent update failure.

```diff
- platforms: ($base.platforms // {} | to_entries + (.platforms // {} | to_entries) | from_entries)
+ platforms: (($base.platforms // {} | to_entries) + (.platforms // {} | to_entries) | from_entries)
```

**2. CSP Blocking Updater Requests (CRITICAL)**

The Tauri app's Content Security Policy only allowed `https://api.github.com` in `connect-src`, but the updater endpoint is on `https://github.com`. Requests to fetch `latest.json` and download update binaries were silently blocked.

**3. Manual-Only Update Installation**

The previous implementation only showed a notification banner requiring manual click on "Download & Restart". Users who dismissed or missed the banner would not receive updates. The updater now automatically downloads and installs updates by default.

| Setting | Before | After |
|---------|--------|-------|
| Initial update check | 30 seconds after launch | **5 seconds** |
| Periodic recheck | Every 4 hours | **Every 1 hour** |
| Window focus recheck | None | **On focus (min 5 min interval)** |
| Update behavior | Manual (banner + click) | **Automatic download & install** |
| Dismiss behavior | Permanent until next 4h cycle | Banner reappears on next check |
| i18n support | English only (hardcoded) | **6 languages** |
| Settings toggle | None | **ON/OFF in Settings page** |

### Release Workflow Hardening

- Platform validation upgraded from `::warning` to `exit 1` — broken `latest.json` will now **fail the release build** instead of being silently uploaded
- Added macOS to platform validation (previously only checked Linux + Windows)
- Updater fragment upload: `if-no-files-found: ignore` → `error`
- Added early empty-platforms check before per-platform validation
