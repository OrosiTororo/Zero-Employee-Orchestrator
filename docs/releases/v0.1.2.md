# v0.1.2 Release Notes

## UI Redesign — VSCode/Zed/Neovim Based

The entire GUI and CLI interface has been redesigned using colors and layout dimensions sourced directly from VSCode (MIT), Zed (Apache 2.0), and Neovim (Apache 2.0).

### Desktop GUI

**Theme System**
- All colors now use exact VSCode Dark Default values (#1E1E1E, #252526, #D4D4D4, #007ACC)
- Status colors (success, error, warning) from Zed editor for better visibility
- 3 built-in themes: Dark, Light, High Contrast
- New Theme Extension API — custom themes can be registered by extensions via `/themes/register`

**Layout**
- Activity Bar: 48px width, 24px icons (VSCode standard)
- Title Bar: 30px height
- Status Bar: 22px height with VSCode blue (#007ACC) background
- Code split: main bundle reduced from 749KB to 388KB (48% reduction, 42 lazy-loaded chunks)

**Welcome Experience**
- First-launch welcome guide on Dashboard with quick-start actions
- Empty state pages now include navigation links back to Dashboard
- 6-language welcome messages (ja, en, zh, ko, pt, tr)

### CLI / TUI

**Neovim-Inspired Mode System**
- NORMAL mode: standard input with slash commands
- INSERT mode: multi-line input (enter with `"""`, exit with `"""` or Ctrl+C)
- COMMAND mode: automatic during slash command execution
- Mode-colored prompt: green (NORMAL), blue (INSERT), yellow (COMMAND)

**Status Line**
- lualine-inspired layout: `NORMAL │ ollama │ ctx:23%    ja │ quality │ ●`
- Dynamic terminal-width-aware padding
- Connection status indicator (● connected / ○ offline)

## Reliability Improvements

**Authentication**
- Token auto-refresh: 401 interceptor retries with refreshed token before logout
- Periodic background refresh every 4 hours prevents session expiry
- Anonymous sessions no longer auto-logout unexpectedly

**Plugin/Extension/Skill Registry**
- 10 built-in plugins and 11 extensions seeded on server startup
- All registry pages migrated to centralized API client (proper auth headers + Tauri URL support)
- "Failed to fetch plugins/extensions" errors resolved

**Setup Wizard**
- Organization build button now shows error messages on failure
- Auto-advances to next step after successful creation
- company_id properly stored for subsequent API calls

## New API

**Theme Extension API**
- `GET /api/v1/themes` — list available themes
- `GET /api/v1/themes/current` — get active theme
- `POST /api/v1/themes/set` — switch theme
- `POST /api/v1/themes/register` — register custom theme with CSS variable overrides
- `GET /api/v1/themes/{slug}/variables` — get CSS overrides for a theme
