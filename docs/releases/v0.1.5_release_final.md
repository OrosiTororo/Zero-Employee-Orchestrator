## 概要

v0.1.5 は Zero-Employee Orchestrator のメジャーリリースです。エンドツーエンドタスク実行エンジン、Copilot Cowork にインスパイアされたタスク管理、ビジュアルワークフロービルダー、スケジュール実行、OpenAPI 完全カバレッジ、セキュリティ強化、Critique パターン/チェックポイント再開などの競合機能を実装しました。

**本バージョンで ZEO は「AI 組織に会社業務を委任する」アーキテクチャとして完成しました。**

---

## 主要新機能

### 1. エンドツーエンドタスク実行エンジン

全 9 層がエンドツーエンドで接続された中核実行パイプラインが完成しました。

- **`POST /tickets/{id}/execute`** — チケット → DAG プラン → LLM 実行 → Judge 検証 → 結果返却
- **`POST /tickets/{id}/generate-plan`** — 仕様書から DAG 実行プラン自動生成
- **`executor.py`** — 全 9 層を接続する中央オーケストレーター (Interview → DAG → LLM → Judge → Repropose → Experience Memory)
- **DAG 並列実行** — 独立した DAG ノードを `asyncio.gather` で並行処理
- **再提案レイヤー** — 失敗分類、プラン差分、信頼スコア付きの自己修復

### 2. Copilot Cowork インスパイア タスク管理

Microsoft Copilot Cowork の plan-approve-execute-steer モデルを参考に実装:

- **プランプレビュー** — `preview_only=true` でコミット前に実行プランを確認。ステップ確認後「承認して実行」
- **ニーズインプットチェックポイント** — LLM が人間入力を必要とするとき自動検出・一時停止。理由メッセージ付きで DispatchPage/CLI から再開
- **実行中ステアリング** — キャンセルなしで追加指示を送り実行方向を変更
- **DispatchPage 再設計** — Active/Done/All タブ、展開可能タスクカード、プランステップ表示、カラーステータスバッジ
- **リッチ承認プレビュー** — アクション内容・リスクレベル (緑/黄/橙/赤) バッジ・コンテキスト表示
- **CLI タスク管理** — 6 つの新スラッシュコマンド: `/dispatch`, `/tasks`, `/status`, `/approve`, `/reject`, `/cancel`

### 3. ビジュアルワークフロービルダー

- **`WorkflowBuilder.tsx`** — パン・ズーム・ドラッグ&ドロップ対応のグラフィカルワークフローキャンバス
- n8n/Zapier のドラッグ&ドロップ体験をインスパイアした実装
- ノード接続、条件分岐、並列実行の可視化

### 4. スケジュール実行

- **`/dispatch/schedules`** エンドポイント — APScheduler cron 統合
- 24/365 バックグラウンド実行
- ビジネスプロセスの定期自動化 (日次レポート、週次集計、月次処理)

### 5. OpenAPI 完全カバレッジ (100%)

- **398/398 エンドポイント** に `response_model` または `response_class` を追加
- OpenAPI 3.1.0 スキーマが完全な型情報を提供
- クライアントコード自動生成が可能に
- 4 エンドポイントは `response_class` (RedirectResponse, FileResponse, HTMLResponse)

### 6. Critique パターン & チェックポイント再開

競合 (Copilot Cowork の Critique パターン、LangGraph のチェックポイント) から採用:

- **Critique パターン** — `enable_critique=True` で第 2 モデルがドラフト出力をレビューしてから採用
- **チェックポイント再開** — `execute_plan()` が `checkpoint_store` dict を受け入れ、長時間プランの一時停止・再開に対応

### 7. セキュリティ強化

- **Kill Switch** — 全 AI 実行の緊急停止メカニズム (`POST /kill-switch/activate`)
- **Judge 階層化** — LIGHTWEIGHT (読み取り) / STANDARD (書き込み) / HEAVY (送信・削除・課金) の 3 ティア (API コスト ~70% 削減)
- **役割ベースツール権限** — 5 つのデフォルトエージェントポリシー (Secretary/Researcher/Reviewer/Executor/Admin)
- **メモリ信頼スコア** — trust ≥ 0.7 かつ期限内のメモリのみ意思決定に使用
- **Red-team 自動テスト** — 8 カテゴリ 22 テスト (プロンプトインジェクション、データ漏洩、権限昇格等)
- **サンドボックスシンリンク強化** — ホワイトリスト外パスへのシンボリックリンク解決をブロック

### 8. AI モデル・プロバイダー強化

- **自動フォールバック** — API キー未設定時に g4f → Ollama へ自動切り替え (「API キー不要」を実現)
- **22 モデルファミリー** / 8 プロバイダー、家族 ID ベースで最新バージョン自動解決
- **5 実行モード** — Quality / Speed / Cost / Free (Ollama) / Subscription (g4f)

### 9. Knowledge Store 強化

- **TF-IDF + コサイン類似度** — LIKE 検索から意味的検索へ
- **LLM 再ランキング** — 検索結果を LLM がスコアリング
- **ベクトルストア抽象化** — 将来的な Pinecone/Qdrant 統合への土台

### 10. Meta-Skills LLM 統合

- `feel()`, `dream()`, `learn()` に実際の LLM 呼び出しを追加 (フォールバック付き)
- `see()`, `make()` は次バージョンで完全 LLM 化予定

## プラットフォーム改善

### デスクトップ自動更新 (CRITICAL バグ修正)

v0.1.2–v0.1.4 ユーザーの自動更新が完全に壊れていた問題を修正:

| 設定 | 修正前 | 修正後 |
|---|---|---|
| 初回更新チェック | 30 秒 | **5 秒** |
| 定期再チェック | 4 時間 | **1 時間** |
| ウィンドウフォーカス時 | なし | **フォーカス時 (最小 5 分間隔)** |
| 更新動作 | 手動バナー | **自動ダウンロード & インストール** |
| i18n | 英語のみ | **6 言語** |

3 つの独立した根本原因を修正:
1. リリースワークフロー jq バグ — `latest.json` の platforms オブジェクトが常に空だった
2. CSP アップデータブロック — `connect-src` が `api.github.com` のみ許可、`github.com` を許可していなかった
3. 手動インストールのみ — デフォルトで自動ダウンロード&インストールに変更

### Judge 深刻度バグ修正 (CRITICAL)

LLM ゲートウェイエラーが `CompletionResponse(content="Error: ...")` を返し、Judge が `no_error_response` ルールの `severity="warning"` (スコア 0.95 > 閾値 0.6) のため通過していた問題:
- `severity="error"` に変更
- executor に `JudgeVerdict.FAIL` チェック追加
- LLM 呼び出し失敗時にタスクが正しく失敗するように

### 組織・Auth 機能

- **Google OAuth** — デスクトップ・ウェブ両対応の完全な OAuth 2.0 フロー
- **組織自動生成** — `POST /org-setup/generate` でビジネスインタビュー回答から部門・チーム・エージェントを一括作成
- **5 ビジネステンプレート** — Content Operations、Sales Research、内部 FAQ/KB、Meeting→Tasks、公開前レビュー
- **SSO/SAML** — エンタープライズシングルサインオン

### UI・UX 改善

- **Cowork スタイルレイアウト** — グループアイコン付きナビバー、中央タイトル、自律ダイヤル + Dispatch フィード付きステータスバー
- **設定 TOC サイドバー** — VSCode スタイルのセクションナビ + 検索フィルタリング
- **オーケストレーション可視化** — 推論トレースタブ (信頼レベル付きタイムライン) + 承認キュータブ (リスク指標・承認/却下ボタン)
- **ダッシュボードチャット履歴** — 自然言語コマンドと応答の永続会話ログ
- **CSS 変数完全化** — ハードコードされた hex カラーを全ページで CSS 変数に置換
