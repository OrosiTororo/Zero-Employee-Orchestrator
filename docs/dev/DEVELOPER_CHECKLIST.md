# Developer Setup Checklist

> Items that require manual configuration (not auto-generated by code).
> Last verified: 2026-04-08

## Quick Start (Minimum for Development)

- [ ] Copy `.env.example` → `.env`
- [ ] Set `SECRET_KEY` (generate: `python -c "import secrets; print(secrets.token_urlsafe(32))"`)
- [ ] Choose ONE LLM provider:
  - Option A: Set `DEFAULT_EXECUTION_MODE=subscription` (uses g4f, no API key needed)
  - Option B: Install Ollama locally (`curl -fsSL https://ollama.ai/install.sh | sh && ollama pull llama3.2`)
  - Option C: Set `GEMINI_API_KEY` (free tier: https://aistudio.google.com/apikey)
  - Option D: Set `OPENROUTER_API_KEY` or `OPENAI_API_KEY` or `ANTHROPIC_API_KEY`

## GitHub Secrets (Required for CI/CD)

| Secret | Workflow | Required? | How to Get |
|---|---|---|---|
| `TAURI_SIGNING_PRIVATE_KEY` | release.yml | Desktop builds | `npx tauri signer generate -w ~/.tauri/signing.key` |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | release.yml | Desktop builds | Set during key generation |
| `CLOUDFLARE_API_TOKEN` | deploy-workers.yml | Edge deploy | Cloudflare Dashboard > API Tokens |
| `CLOUDFLARE_ACCOUNT_ID` | deploy-workers.yml | Edge deploy | Cloudflare Dashboard > Overview |
| `CLAUDE_CODE_OAUTH_TOKEN` | claude.yml | @claude in PRs | Claude Code OAuth flow |
| `SECRET_KEY` | deploy-api.yml | API deploy | `python -c "import secrets; print(secrets.token_urlsafe(32))"` |
| `DEPLOY_HOST` / `DEPLOY_USER` / `DEPLOY_SSH_KEY` | deploy-api.yml | VPS deploy | Your server SSH config |
| `FLY_API_TOKEN` | deploy-api.yml | Fly.io deploy | `flyctl tokens create deploy` |

**PyPI**: Uses OIDC trusted publisher (no token needed if configured at pypi.org).

## GitHub Repository Settings

- [ ] Branch protection on `master` (require PR review, pass CI)
- [ ] Enable secret scanning (Settings > Security)
- [ ] Create `production` environment with approval requirement
- [ ] Dependabot enabled (already configured in `.github/dependabot.yml`)

## Chrome Extension (Browser Assist)

**Local testing:**
1. Open `chrome://extensions`
2. Enable "Developer mode"
3. Click "Load unpacked"
4. Select `extensions/browser-assist/chrome-extension/`

**Chrome Web Store publishing (when ready):**
- [ ] Create developer account ($5 one-time fee): https://chrome.google.com/webstore/devconsole
- [ ] Prepare screenshots (1280x800 or 640x400)
- [ ] Privacy policy URL: `https://github.com/OrosiTororo/Zero-Employee-Orchestrator/blob/master/PRIVACY_POLICY.md`
- [ ] Submit extension for review

## Production Deployment

- [ ] Set `DEBUG=false`
- [ ] Change `SECRET_KEY` to a unique value
- [ ] Switch `DATABASE_URL` to PostgreSQL
- [ ] Update `CORS_ORIGINS` to actual domain
- [ ] Run `./scripts/security-check.sh`
- [ ] Configure HTTPS (reverse proxy or cloud provider)
