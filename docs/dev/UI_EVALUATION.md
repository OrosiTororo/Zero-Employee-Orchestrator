# ZEO UI 評価レポート v2 (2026-04-06)

## 評価基準の定義

今後の評価は以下の視点で行う。この基準は `docs/dev/UI_EVALUATION.md` に記録し、
継続的に参照する。

| 視点 | 定義 | 重み |
|------|------|------|
| **相対的評価** | VSCode/Zed等の競合IDEとの比較。カラー準拠度、レイアウト一貫性、VSCodeユーザーの習得容易さ | 25% |
| **客観的評価** | AIオーケストレーターとして初見ユーザーの視点。何ができるか分かるか、迷わないか、説明が十分か | 30% |
| **技術的評価** | ビルド品質、CSS変数整合性、バンドルサイズ、パフォーマンス、アクセシビリティ | 20% |
| **ZEO方針適合度** | 「軽量IDE」「拡張機能で追加」「VSCodeベース」「CLI=Neovim風」の方針への適合 | 15% |
| **CLI評価** | Neovimインスパイアの度合い。ステータスライン、モード切替、操作効率 | 10% |

---

## 1. 相対的評価（競合比較）: 9.0/10

### カラー準拠度: 9.5/10
- VSCode Dark Default (#1E1E1E, #252526, #D4D4D4, #007ACC) を全面採用（MIT）
- ステータスカラーはZed (#56BA9F, #E5484D) を採用（Apache 2.0、視認性向上）
- `--accent-secondary: #7C3AED` はVSCode非公式だがZed系統。機能上必要
- 全3テーマ（dark/light/high-contrast）で全変数一貫定義

### レイアウト一貫性: 9.0/10
- Activity Bar 48px, Title Bar 30px, Status Bar 22px — VSCode実ソースと完全一致
- Tab Barは削除（AIオーケストレーターにファイルタブは不要 — 合理的判断）
- Command Palette (Ctrl+K) がVSCodeと同じUXを提供

### 習得容易さ: 8.5/10
- VSCodeユーザーは即座に操作理解可能（同一レイアウト・同一ショートカット）
- SettingsページにVSCode式TOCサイドバー（8セクション）実装済み
- 全UIテキストがi18nキー化（6言語完全対応）

---

## 2. 客観的評価（初見ユーザー視点）: 8.0/10

### 何ができるか分かるか: 8.0/10
- ウェルカムガイド: 初回表示で「自然言語で業務を説明 → AIチームが実行」と明示
- 5つのクイックスタートテンプレートが具体例を提供
- 秘書ページへの直接リンクが「まず何をすべきか」を明確化

### 迷わないか: 8.0/10
- ログイン → セットアップ → ダッシュボードの流れが明確
- Command Palette (Ctrl+K) で全ページにアクセス可能
- 空状態ページにダッシュボードへのナビゲーションリンク追加済み
- **改善余地**: Activity Barのアイコンのみでの機能推測（ツールチップはある）

### 操作の一貫性: 8.0/10
- 全ページがCSS変数ベースの統一テーマ
- フォームスタイル統一（入力フィールド、ボタン、エラー表示）
- 全6言語でウェルカムガイド翻訳済み

## 3. 技術的評価: 9.0/10

### ビルド品質: 9.5/10
- TypeScript: エラーなし
- ruff: エラーなし
- Vite build: 成功（2.92s）
- デモサーバー: 全ページ200 OK

### CSS変数整合性: 10/10
- 全3テーマで全変数定義済み（--success-fg, --warning-fg含む）
- 41箇所の未定義変数問題を全て修正済み
- ハードコード色をCSS変数に全置換済み

### バンドルサイズ: 9.0/10
- **コードスプリット導入**: 749KB → 388KB（メインバンドル48%削減）
- 42個の個別チャンクに分割
- CSS: gzip 8KB
- **改善余地**: i18nバンドル（59KB gzip）の遅延ロード

### パフォーマンス: 9.0/10
- dev server: 622msで起動
- 各ページはlazy loadでオンデマンドロード
- Suspense fallbackでロード中表示

### アクセシビリティ: 7.5/10
- focus-visible, aria-label, aria-current
- **改善余地**: aria-live、ランドマーク、キーボードフォーカス順序

## 4. ZEO方針適合度: 9.0/10

### 「軽量IDE」: 9.0/10
- Tab Bar削除、カスタムシャドウ/グラデーション全削除で視覚的に軽量
- コードスプリットで初回ロードが軽量
- 不要な視覚装飾を排除

### 「拡張機能で追加」: 8.5/10
- **テーマ拡張API実装**: `/themes/register` で拡張機能からCSS変数オーバーライド可能
- `/themes/set` でアクティブテーマ切替
- ステータスカラーはCSS変数で上書き可能
- マーケットプレイスは「Coming Soon」（v0.2ロードマップ通り）

### 「VSCodeベース」: 9.5/10
- カラー: VSCode Dark Default 完全準拠（MIT）
- 寸法: VSCodeソースから取得した公式値
- UIパターン: Activity Bar, Command Palette, Status Bar, TOC Settings

---

## 5. CLI評価（Neovim inspired）: 8.5/10

### ステータスライン: 9.0/10
- lualine風: `NORMAL │ ollama │ ctx:23%    ja │ quality │ ●`
- VSCodeアクセントカラー使用
- 動的幅計算（ターミナル幅に応じてパディング調整）

### モード切替: 8.5/10
- **NORMAL**: 通常入力。スラッシュコマンド可
- **INSERT**: `"""`で入り、複数行入力。`"""`で抜ける。Ctrl+Cで NORMAL復帰
- **COMMAND**: `/`で自動遷移、実行後 NORMAL復帰
- モードに応じたプロンプト色変化（緑/青/黄）

### バナー: 8.0/10
- VSCode/Zedカラー統一
- Neovim lualineステータスライン統合
- **改善余地**: checkhealth風の依存状態チェック表示

## ユーザー追加要件の実現可能性

### 1. 「フォルダにナレッジを入れると秘書AIが把握」
- **状態**: 実装済み（knowledge_store.py + secretary routes）
- KnowledgeRecordモデルでファイル/フォルダのナレッジを永続保存
- ユーザーはチャットで「このフォルダを読み込んで」と指示可能

### 2. 「GPTとGeminiに同じ質問をして比較」
- **状態**: 実装済み（multi_model.py + brainstorm routes）
- `POST /brainstorm` で複数モデル同時比較セッション作成
- ユーザーはチャットで「GPTとGeminiに聞いて」と指示可能（NLコマンド）

### 3. 「やり取りは全て自動保存」
- **状態**: 実装済み（BrainstormSessionRecord + conversation_history）
- セッションベースで会話履歴を永続保存
- Experience Memoryで過去の文脈を踏まえた回答

### 4. 「ユーザーがAI組織とのチャットのみで機能追加」
- **状態**: 実装済み（nl_command_service.py）
- 「スキルを追加して」→ `SKILL + INSTALL` コマンド実行
- 「プラグインを作って」→ `SKILL + GENERATE` コマンド実行
- `POST /registry/skills/generate` で自然言語からスキル自動生成
