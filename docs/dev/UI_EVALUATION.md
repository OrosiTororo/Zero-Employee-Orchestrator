# ZEO UI 評価レポート (2026-04-06)

## 評価基準

本レポートでは以下の視点から評価を行う：

1. **相対的評価** — VSCode/Zed等の競合IDEとの比較。カラー準拠度、レイアウト一貫性、習得の容易さ
2. **客観的評価** — AIオーケストレーターとして初見ユーザーの視点。何ができるか分かるか、迷わないか
3. **技術的評価** — ビルド品質、CSS変数の整合性、パフォーマンス、アクセシビリティ
4. **ZEO方針適合度** — 「軽量IDE」「拡張機能で追加」「VSCodeベース」の方針に沿っているか

---

## 1. 相対的評価（競合比較）

### カラー準拠度: 9/10
- VSCode Dark Default のカラーをMITライセンスで直接採用
- ステータスカラーはZed (Apache 2.0) から採用し、視認性向上
- **減点**: `--accent-secondary: #7C3AED`（紫）はVSCode公式には存在しない独自追加。ただしZedのaccent-secondaryに近く、機能的に必要

### レイアウト一貫性: 8/10
- Activity Bar 48px, Title Bar 30px, Status Bar 22px — VSCode実ソースと完全一致
- Tab Barは客観的判断で削除（AIオーケストレーターにファイルタブは不要）
- **減点**: サイドバー展開機能はVSCodeにない独自機能。ただしZEOのページ数（17+ページ）を考慮すると有用
- **減点**: VSCodeのBreadcrumb（パス表示）がない。ただしZEOではページ階層が浅いため不要

### 習得の容易さ: 8/10
- VSCodeユーザーなら操作を即理解（Activity Bar, Command Palette Ctrl+K, Status Bar）
- ウェルカムガイドが初回表示される
- **減点**: 設定ページのVSCode式TOCサイドバー（CLAUDE.md要件）が現状実装されているか未確認

---

## 2. 客観的評価（初見ユーザー視点）

### 何ができるか分かるか: 7/10
- ダッシュボードにタスク入力欄と4つのクイックアクション
- ウェルカムガイドが「自然言語で業務を説明 → AIチームが実行」と説明
- 5つのテンプレートが具体例を提供
- **問題点**: 「AI秘書」「壁打ち」「エージェント監視」等の用語が初見で分かりづらい
- **問題点**: Activity Barのアイコンだけでは機能が推測しにくい（ホバーツールチップはあるが発見しにくい）

### 迷わないか: 7/10
- ログイン → セットアップ → ダッシュボードの流れは明確
- Command Palette (Ctrl+K) で全ページに素早くアクセス可能
- **問題点**: 初回ドキュメント表示がまだREADME.mdの内容を直接表示しない（ウェルカムバナーのみ）
- **問題点**: 空状態（チケットなし、エージェントなし）のときの誘導が弱い

### 操作の一貫性: 8/10
- 全ページがCSS変数ベースの統一テーマ
- フォームスタイルが統一（入力フィールド、ボタン）
- **改善余地**: 一部ページ（BrainstormPage, AgentMonitorPage）のインラインスタイルが微妙に異なる

## 3. 技術的評価

### ビルド品質: 9/10
- TypeScript: エラーなし
- ruff: エラーなし
- Vite build: 成功 (2.04s)
- CSS: 36KB (gzip 8KB) — 軽量
- **減点**: メインバンドル 749KB (gzip 207KB) — コードスプリットが必要

### CSS変数の整合性: 9/10
- 全3テーマ（dark/light/high-contrast）で全変数定義済み
- **修正済み**: 41箇所の未定義変数参照を修正（--success-fg, --warning-fg, --accent-subtle, --accent-secondary追加）

### パフォーマンス: 8/10
- 初回ロード: dev serverで376msで起動
- ビルドサイズは適切（CSS 8KB gzip）
- **改善余地**: JSバンドルの分割（ダイナミックインポート）

### アクセシビリティ: 7/10
- focus-visible outlineあり
- aria-labelがActivity Barボタンに設定
- role="tablist" は Tab Bar 削除に伴い不要に
- **改善余地**: スクリーンリーダー対応（aria-live、ランドマーク）が不十分
- **改善余地**: キーボードナビゲーション（Tab/Shift+Tab）のフォーカス順序

---

## 4. ZEO方針適合度

### 「軽量IDE」: 8/10
- Tab Bar削除、カスタムシャドウ/グラデーション全削除で視覚的に軽量
- CSS変数ベースで拡張機能によるテーマ変更が可能
- **改善余地**: SettingsページにVSCode式TOCサイドバーが必要（CLAUDE.md要件）

### 「拡張機能で追加」: 7/10
- カラーテーマはCSS変数で拡張機能から変更可能
- ステータスカラーはCSS変数で上書き可能
- **問題点**: 拡張機能でテーマを変更するAPI/UIがまだ実装されていない
- **問題点**: 拡張機能マーケットプレイスページが「Coming Soon」のまま

### 「VSCodeベース」: 9/10
- カラー: VSCode Dark Default完全準拠
- 寸法: VSCodeソースから取得した公式値
- UI パターン: Activity Bar, Command Palette, Status Bar

---

## 5. CLI評価（Neovimインスパイア）

### ステータスライン: 8/10
- lualine風のセクション分割（mode | provider | ctx  lang | mode | status）
- VSCodeアクセントカラー使用
- **改善余地**: 動的更新（応答中のモード変更表示）がまだない

### バナー: 8/10
- ASCII artロゴ維持
- カラーをVSCode/Zedパレットに統一
- **改善余地**: Neovimのcheckhealth風の依存状態チェック表示

### 操作感: 7/10
- スラッシュコマンドが充実（/read, /write, /edit, /run等）
- **改善余地**: Neovimのモード切替（Normal → Insert → Command）が表示のみで実動作しない
- **改善余地**: ヒストリーナビゲーション（↑↓キー）のTUI対応
