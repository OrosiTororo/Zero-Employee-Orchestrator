あなたは CommandWeave リポジトリで作業する Claude Code です。

まず最初に、次の3ファイルを読んでください。

1. `CLAUDE.md`
2. `IMPLEMENTATION_PLAN.md`
3. `CommandWeave.md`

読み取り後は、思想文書を全部一気に実装しようとせず、`CLAUDE.md` と `IMPLEMENTATION_PLAN.md` を最優先の基準にしてください。

今回の目的は、CommandWeave の MVP として以下の end-to-end を最小構成で通すことです。

- ユーザーが自然言語で依頼を入力する
- Design Interview で要件を深掘りする
- `spec / plan / tasks` を生成・保存する
- 計画にコスト、権限、危険操作、担当要素を表示する
- ユーザーが承認または差し戻しできる
- 承認後に実行へ進む
- Judge で品質確認する
- 成果物、実行ログ、承認履歴を保存する

作業ルール:

- 一度に大きく壊さない
- 既存構造を無視して新構造を乱立させない
- 承認必須操作を自動実行前提で実装しない
- 本体に業務固有ロジックを過剰に埋め込まない
- 安全性、監査可能性、権限表示を優先する
- 見た目だけでなく end-to-end が通ることを優先する

最初にやること:

1. 現在のリポジトリ構造を確認する
2. 既存の入力導線、保存構造、実行導線、UI / API 構造を把握する
3. `IMPLEMENTATION_PLAN.md` の Phase 0 観点で不足を列挙する
4. 変更対象ファイルと最小実装方針を提示する
5. その後、最小差分で実装を開始する

出力フォーマット:

最初の返答では必ず次を出してください。

- 現状把握の要約
- 実装上の不足点
- 変更対象候補ファイル
- 最初の実装ステップ
- 今回はやらないこと

実装後は毎回次を明示してください。

- 何を変更したか
- なぜ変更したか
- どの受け入れ条件を満たしたか
- 未完了の項目は何か
- 次の最小ステップは何か

受け入れ条件:

- 自然言語入力からジョブを作成できる
- `spec / plan / tasks` が保存される
- 計画に権限・危険操作・コスト概算が表示される
- 承認または差し戻しできる
- 高リスク処理は承認なしに進まない
- 実行ログが残る
- Judge 結果が残る
- 成果物と履歴を再確認できる

迷った場合は、より小さく、安全で、説明可能な案を選んでください。
